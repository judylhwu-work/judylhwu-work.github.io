Nova Design System: Figma to Claude Code

With Nova as the foundation, I built two AI applications to scale design system expertise and collapse the handoff gap between design and engineering.

Role Senior Product Designer, Platform Team
Duration 2025 — Present
Team Solo (design + build)
Tools Claude API, Figma, HTML, CSS, JavaScript

Ask Nova

The problem

I was the single point of contact for all Nova-related questions — in Slack, in Figma comments, in reviews. As the design system matured and more designers and engineers used it, the volume of repeated questions grew. How do I use this token? Which component should I reach for? Where do I find X in the library? My time was getting carved up answering the same questions across different people and contexts.

What I built

Ask Nova is an AI design system chatbot built on Claude, grounded in Nova's documentation, design tokens, and component logic. Designers and engineers can ask it natural language questions and get answers immediately — without waiting for me. Ask Nova knows the system: the tokens, the component rationale, the intended usage patterns.

How it handles edge cases

One of the core design challenges was knowing when the AI shouldn't answer. I built a confidence-score system: Ask Nova evaluates its certainty before responding. High-confidence questions get answered directly. Low-confidence questions — anything ambiguous, highly contextual, or outside Nova's documented scope — escalate to me instead. The system scales my time without replacing my judgment.

Impact

Routine design system questions are now handled without me in the loop. Engineers and designers get answers in seconds instead of waiting for my availability. I spend that recovered time on the work that actually requires my context — designing, reviewing, and evolving the system itself.

HTML Editor Tool

The problem

A PM asked me to help fix a UI issue in a table component. As I looked into it, I realized the gap wasn't just about that specific component — designers at LeanTaaS had no direct way to work in HTML. All frontend changes went through engineering, creating a bottleneck where small UI corrections accumulated into a backlog. The design-to-dev handoff was structurally broken for the kind of incremental, precision-heavy work that makes a design system actually land well in production.

What I built

The HTML Editor Tool is an AI-powered editor that lets designers work directly in HTML using natural language. Instead of speccing changes in Figma and hoping they get implemented correctly, a designer can describe what they want and see it update in a live preview. The tool bridges the gap between design intent and frontend implementation — without requiring designers to learn to code.

Reception

When I showed the PM, they flagged resourcing concerns. They hadn't expected design to build this kind of tooling. That reaction told me two things: the problem was real enough that engineering's involvement seemed obvious in hindsight, and design-initiated infrastructure was genuinely unexpected at LeanTaaS.

Status

The HTML Editor Tool is in active development. It's part of a larger effort to give design more direct leverage over how Nova components look and behave in production — closing the loop between the design system and the shipped product.
